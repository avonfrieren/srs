# srs — Plan d'implémentation

Feuille de route validée (2026-07-18, découpage phase 4 confirmé : comparaison d'abord, auto-détection ensuite). Projet issu du point 8 (phase 3) du `PLAN.md` de srta : les temps colorés par comparaison deviennent un mod à part entière, connecté à SpeedrunTool (dépendance, comme srta).
But : importer la practice sheet (Google Sheets → CSV local), comparer le temps final d'un segment du room timer de SpeedrunTool aux seuils de la sheet, et afficher la couleur du palier atteint.
Voir `CLAUDE.md` pour l'architecture et les décisions techniques.

## Phase 0 — Prérequis — ✅ faite (2026-07-18)

1. ~~**Accord du créateur de la sheet**~~ — condition héritée du PLAN.md de srta, confirmée par le propriétaire du projet.
2. ~~**Audit de la structure de la sheet**~~ — onglet « Celeste Any% Standars CP'S » (`gid=639470957`) : les paliers sont des **colonnes nommées par couleur** (Hidden, WR, Gold, Pink, Purple 1-3, Indigo 1-3, Blue 1-3, Cyan 1-3, Green 1-3, Olive 1-3, Yellow 1-3, Orange 1-3, Red 1-3, Unranked) → l'export CSV suffit, **pas besoin de xlsx**. Deux blocs : segments (Prologue, 1a…7a, dont `5b` et `6a Tape`) puis « Chapter Times » (+ `Filetime Buffer`, `Sob`). Formats de temps mixtes : `28`, `28.1`, `00:56`, `1:05.5`, `24:06.802`.
3. **Mapping paliers → couleurs RGB** — les noms de colonnes donnent la couleur ; les valeurs RGB exactes seront saisies à la main plus tard (phase 4).

## Phase 1 — Squelette du mod — ✅ faite (v0.1.0)

4. ~~**Repo `srs` sur le modèle de srta**~~ — `srs.csproj` (extraction de `SpeedrunTool.dll` du zip installé, cible `OutputAsModStructure`), `everest.yaml` (dépendances Everest + SpeedrunTool 3.27.16), `SrsModule`, `SrsSettings`, `Dialog/` EN+FR. Pas de Publicizer pour l'instant (aucun internal SpeedrunTool utilisé ; à ajouter en phase 4 si l'interop ne suffit pas).

## Phase 2 — Import & données — ✅ faite (v0.1.0)

5. ~~**Téléchargement CSV**~~ — GET HTTP sur `https://docs.google.com/spreadsheets/d/<id>/export?format=csv&gid=<gid>` (sheet publique « toute personne disposant du lien » : aucun compte, aucune clé, aucun credential stocké). URL configurable (`SheetUrl` dans les settings, `[SettingIgnore]`), id/gid extraits par regex.
6. ~~**Parseur & modèle**~~ — parseur CSV RFC 4180 + parseur de temps `[hh:][mm:]ss[.fff]` ; modèle `SheetData` → blocs → segments → temps alignés sur les colonnes de couleur. Jamais d'exception sur contenu malformé (cellules illisibles ⇒ null).
7. ~~**Cache local & fallback manuel**~~ — CSV cachés dans `Saves/srs/sheet.csv` (écriture atomique), rechargé au démarrage ⇒ le mod marche hors-ligne. Déposer un CSV exporté à la main au même endroit = import manuel (cas sheet privée). Bouton « Mettre à jour » dans Mod Options (téléchargement async, jamais au lancement du jeu), statut (nb de segments + date) affiché sous le bouton, erreurs dans log.txt.

## Phase 3 — Sélection du checkpoint — ✅ faite (v0.2.0)

8. ~~**Menu de sélection**~~ — clarification 2026-07-18 : l'unité à sélectionner est le **checkpoint** (ex. 1a → 1a Start, Crossing, Chasm), pas le chapitre entier (ça, ce sont les IL). L'onglet importé devient « Celeste Any% Standards CP's » (`gid=276158492`) : tous les checkpoints de la route any%, y compris les deux choix de route (`5a/b`, `6a`, `6a Route`, `6b Route`) — 41 checkpoints en 10 groupes + bloc « Chapter Times ». Deux sliders dépendants dans Mod Options (Chapitre → Checkpoint, liste reconstruite au changement de chapitre), sélection persistée par noms `(chapitre, checkpoint)` (les noms seuls se répètent : « Wake Up », « Rock Bottom »). Migration auto de l'ancienne `SheetUrl` par défaut (onglet IL) ; les CSV chapitre-seuls restent lisibles (fallback : chaque chapitre = son unique checkpoint). La même architecture (bloc → chapitres → checkpoints) servira telle quelle aux futures catégories. **Auto-détection depuis `Session.Area` reportée en phase 4** (elle se décidera avec le hook du room timer ; ambiguë au niveau checkpoint côté menu).

## Phase 4 — Comparaison & affichage — ✅ faite (v0.3.0)

9. ~~**Comparaison au timer SpeedrunTool**~~ — à `RoomTimerIsCompleted()` (ModInterop `SpeedrunTool.RoomTimer`), le temps est **capturé sur la frame de complétion** (découverte à l'implémentation : `GetRoomTime()` = `Data_Auto.Time` continue de courir après complétion, seul l'affichage gèle via `ThisRunTimes`) puis comparé aux seuils du segment sélectionné : premier seuil ≥ temps ⇒ palier atteint ; au-delà de Red 3 ⇒ Unranked ; seuils null (cellules vides) et `Hidden` (0:00 partout) ignorés. Le palier est recalculé chaque frame depuis le temps capturé ⇒ réagit aux changements de sélection/re-imports ; seuls `wasCompleted` + `capturedTicks` sont enregistrés via l'interop `SpeedrunTool.SaveLoad`.
10. ~~**HUD couleur**~~ — nom du palier dans sa couleur, ligne sous le timer (après `orig` de `SpeedrunTimerDisplay.Render`, même fond/animation que les deltas de srta ; **décalée d'une ligne si srta est chargé** pour ne pas chevaucher sa ligne de delta). Couleurs résolues par réflexion sur les propriétés statiques de `Microsoft.Xna.Framework.Color` depuis les noms de colonnes (suffixes « 1-3 » retirés, vérifié : les 11 noms existent dans FNA) ; `WR`/`Hidden` ⇒ blanc, `Unranked` ⇒ gris. Toggle `ShowTier` dans Mod Options (menu auto-généré).
11. ~~**Détection de fin de run par nombre de rooms**~~ — gap découvert au premier test en jeu (2026-07-18) : `RoomTimerIsCompleted()` ne se déclenche qu'au « Number of Rooms » **réglé à la main dans SpeedrunTool** — srs ne savait pas quand un run du checkpoint sélectionné était réellement fini (palier après 1 room au réglage par défaut, ou jamais si le timer est Off ; en plus le `srs.zip` installé datait d'avant `TierComparison`). Fix : table des rooms par checkpoint (`RoomCounts.cs`, données communautaires fournies à la main), **poussée dans `NumberOfRooms` à chaque changement de sélection** (le run part du début du checkpoint ⇒ le timer se termine pile sur sa dernière room) ; à la complétion, capture seulement si `NumberOfRooms` == la valeur attendue (override manuel = practice partielle ⇒ pas de palier ; rouvrir le menu sans bouger la sélection ne re-pousse pas la valeur). Checkpoints à rooms inconnues (« ? » : les finaux de chaque chapitre, Granny, Hollows Tape) ⇒ `NumberOfRooms` = 99 (max du slider SpeedrunTool) : le timer ne se termine que par ce qui finit vraiment le run (fin de chapitre, cassette, drapeau du Sommet). Rooms encore inconnues : Chasm, Awake, Presidential Suite, Cliff Face, Mix Master, Resolution, Reprieve, 3000m, Hollows Tape (+ Unravelling 12 / Search 8 notées en commentaire : absentes de l'onglet actuel, qui ne couvre que la route 5B).

## Phase 4bis — Auto-détection du segment — ✅ faite (v1.0.0)

- ~~**Chapitre**~~ : `Session.Area` (`AreaKey.ID` 0=Prologue…7=Summit + `Mode` Normal/BSide) → chapitre sheet, y compris le côté des chapitres pliés (ID 5 + BSide ⇒ « 5a/b » côté `5b` ; ID 6 quel que soit le mode ⇒ « 6a/b »). Table `ChapterMap` dans `SegmentAutoDetect`.
- ~~**Checkpoint**~~ : `AreaData.Get(area).Mode[mode].Checkpoints` (`CheckpointData.Level` = room de départ, `Name` = clé dialog `CHECKPOINT_6_2` → `Dialog.Clean` **forcé en anglais** — les clés sont traduites, le mapping ne doit pas dépendre de la langue du joueur) + `Session.StartCheckpoint` (room du checkpoint choisi au chapter panel, null = départ du début ⇒ « Start » ; correction d'exploration : `Session.FirstLevel` est un bool, pas la room de départ). Suivi vivant : initialisé à `Everest.Events.Level.OnEnter`, mis à jour à chaque transition vers une room de checkpoint (`LevelData.HasCheckpoint`), état statique enregistré via `RegisterStaticTypes` ⇒ charger un savestate restaure le checkpoint du moment de la sauvegarde ; la sélection est réappliquée chaque frame (pas de callback de load côté SpeedrunTool).
- ~~**Noms jeu ↔ sheet**~~ — **décision propriétaire (2026-07-18) : pas de normalisation, table hardcodée** (`CheckpointMap`) : la sheet actuelle est un prototype, sa successeure sera sensiblement différente et exigera de toute façon un nouveau parseur. Divergences couvertes par la table : `500 M` → `500m` (etc.), `Through the Mirror` → `Through The Mirror`, 6B `Reflection` → `Falling`, « Start » par côté (`5a Start`…), `Rock Bottom` re-préfixé. `Granny` = le « Start » du Prologue. `Hollows Tape` reste manuel (ambigu avec `Hollows`) ; les checkpoints jeu absents de la sheet (route 5A complète après Depths) ne changent pas la sélection.
- ~~Toggle Auto/Manuel~~ : `AutoDetect` (défaut on), entrée manuelle dans le menu qui grise les deux sliders (`TextMenu.Item.Disabled`) ; sliders = override manuel. Chaque bascule de sélection auto pousse aussi le Number of Rooms (`RoomCounts.Apply`), comme les sliders ; le hook `Level.Update` de l'auto-détection est **externe à celui de `TierComparison`** (chargé après) pour que la capture d'une complétion voie encore la sélection du début du run, et la détection est **suspendue tant que le timer est complété** (découvert au test en jeu : finir un run transitionne dans la room du checkpoint suivant ⇒ sans ça la sélection re-ciblait Number of Rooms, dé-complétait le timer et cachait le résultat) — elle rattrape au reset du timer (load de savestate, clear).

## Phase 5 — Finitions — ✅ faite (v1.0.0)

11. ~~**Hotkey toggle de l'affichage**~~ — `ButtonBinding` (`ToggleShowTier`, non lié par défaut, config clavier/manette d'Everest), géré dans le hook `Level.Update` de `TierComparison` avec le popup on/off de SpeedrunTool (`PopupMessageUtils`, classe publique — toujours pas de Publicizer) ; textes EN/FR complets, README changelog, v1.0.0.

## Écarté

- **Sheets privées (OAuth / service account)** — hors périmètre définitif : credentials à stocker chez le joueur, API lourde et fragile. Le fallback = export CSV manuel déposé dans `Saves/srs/`.
- **Téléchargement automatique au lancement** — refusé : import uniquement à la demande (bouton), le cache local fait le reste.
